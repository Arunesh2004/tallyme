# Roadmap B1.0 — Enterprise Product Foundation

## 1. Product Development Roadmap
This roadmap transitions TallyMe from an architectural proof-of-concept into a production-ready enterprise product. The core objective is validating the architecture via end-to-end functionality including authentication, organization management, document processing, and viewing exports.

Phase B1 modules:
- Authentication & RBAC
- Organization & Workspace Management
- Document Management & AI Pipeline Dashboard
- Business Document & Export Package Viewers
- Integration Configuration & Global Settings

## 2. Updated Folder Structure
The architecture adheres to DDD and Clean Architecture on the backend, and Feature-Sliced Design or App Router organization on the frontend.
```text
tallyme/
├── apps/
│   ├── web/ (Next.js 15, React 19)
│   │   ├── app/ (Routing, Pages, Layouts)
│   │   ├── components/ (Shared UI)
│   │   ├── features/ (Domain-specific components & hooks)
│   │   ├── lib/ (API clients, utilities)
│   │   └── types/ (TypeScript models)
│   └── api/ (FastAPI)
│       ├── application/ (Use Cases, DTOs, Services)
│       ├── domain/ (Entities, Value Objects, Events)
│       ├── infrastructure/ (Persistence, DB, ORM)
│       ├── presentation/ (REST Controllers)
│       └── integrations/ (Provider Placeholders)
```

## 3. Frontend Architecture
- **Framework**: Next.js 15 (App Router) + React 19.
- **Language**: TypeScript (Strict Mode).
- **Styling**: TailwindCSS with shadcn/ui. Dark mode default, responsive first.
- **Data Fetching & State**: TanStack Query (React Query) for server state.
- **Forms**: React Hook Form + Zod for validation.
- **Animations**: Framer Motion for micro-interactions and transitions.

## 4. Backend API Architecture
- **Framework**: FastAPI (Python).
- **Design**: Thin REST controllers. No business logic in presentation layer.
- **Flow**: Controllers -> Application Services -> Use Cases -> Domain.
- **Serialization**: Pydantic models (DTOs) in Application layer. Domain objects remain pure.

## 5. Authentication Design
- **Strategies**: JWT-based stateless auth for API (access + refresh tokens).
- **Features**: Registration, Login, Logout, Forgot Password, Profile Management.
- **RBAC**: Owner, Admin, Manager, Member, Viewer.
- **Session Management**: Handled via secure HttpOnly cookies where applicable, and Next.js Auth.js or custom context.

## 6. Organization Model
- **Core Entity**: `Organization`.
- **Associations**: `Members` (with specific RBAC roles), `Invitations`.
- **Attributes**: Branding settings, Audit Log, Subscription/Quota limits.

## 7. Workspace Model
- **Core Entity**: `Workspace` (belongs to Organization).
- **Associations**: Contains `Documents`, `BusinessDocuments`, `ExportPackages`, `Integrations`.
- **Purpose**: Logical segregation of workflows within an organization (e.g., HR vs Finance workspaces).

## 8. Dashboard Architecture
- **Home Dashboard**: Displays processing queues, recent uploads, and pending documents.
- **Statistics**: Organization metrics, export stats, and integration statuses.
- **Real-time**: Optimistic updates using TanStack Query.

## 9. Document Management Architecture
- **Upload Flow**: Drag & drop support. Direct to S3/blob storage, dispatching an event to the AI Pipeline.
- **Features**: Versioning, Tags, Search, Status filtering.
- **Lifecycle**: Upload -> Pipeline -> Processed/Failed.

## 10. Business Document Viewer Design
- **Presentation**: Dual-pane view (Document preview + Extracted Data).
- **Data Displayed**: `BusinessFacts`, Entities, Financial & Tax Summaries, Line Items, Metadata.
- **UX**: Highlights mapping document regions (Confidence scores, Lineage tracking).

## 11. Export Package Viewer Design
- **Presentation**: Details of `ExportPackage`, `ExportProfile`, and `ExportPolicy`.
- **Transparency**: Shows Artifact Metadata, Constraints, Manifest, Validation Results.
- **Rule**: No provider-specific adapter data displayed; purely the canonical boundary representation.

## 12. Integration Management Design
- **Functionality**: Enable/disable plugins, manage credentials securely.
- **Views**: Synchronization history, permission mapping.
- **Current State**: Placeholders only. No concrete ERP implementations (e.g., NetSuite/Xero) yet.

## 13. REST API Specification
Prefix: `/api/v1/`
- `POST /auth/login`, `POST /auth/register`
- `GET /organizations`, `POST /organizations`
- `GET /workspaces/{id}`, `GET /workspaces/{id}/documents`
- `POST /documents/upload`
- `GET /business-documents/{id}`
- `GET /export-packages/{id}`
- `GET /integrations`

## 14. Navigation Map
- **Sidebar**: Dashboard, Documents, Processing, Exports, Integrations, Settings.
- **Top Bar**: Global Search, User Profile, Workspace Switcher, Notifications.

## 15. Component Hierarchy
- `AppLayout` -> `Sidebar`, `Header`, `MainContent`
- `MainContent` -> `PageContainer` -> `FeatureComponent`
- Example (`DocumentViewer`): `SplitPane` -> `PDFPreview` + `ExtractedDataSidebar`.

## 16. State Management Strategy
- **Server State**: TanStack Query (caching, invalidation, polling for processing status).
- **Form State**: React Hook Form.
- **UI State**: React Context or Zustand for simple globals (sidebar toggle, theme).

## 17. Security Architecture
- **API**: Bearer tokens, CORS strictly configured.
- **Data**: TLS in transit, AES-256 at rest.
- **Domain**: PII redaction pipeline before LLM processing.
- **RBAC**: Enforced at the Use Case level, not just the controller.

## 18. Testing Strategy
- **Unit**: Pytest for backend domain/application, Vitest for frontend utilities.
- **Integration**: API tests against test DB, React Testing Library for feature flows.
- **E2E**: Playwright for critical paths (Registration -> Upload -> View Document).

## 19. Deployment Recommendations
- **Frontend**: Vercel or Containerized Node.js.
- **Backend**: Containerized FastAPI on ECS/EKS.
- **Database**: Managed PostgreSQL.
- **Storage**: Amazon S3.

## 20. MVP Milestone Plan
- **M1 (Weeks 1-2)**: Auth, Organization, Workspace setup (Backend + Frontend scaffolding).
- **M2 (Weeks 3-4)**: Document Upload and Processing Dashboard UI.
- **M3 (Weeks 5-6)**: Business Document Viewer and Export Package Viewer.
- **M4 (Weeks 7-8)**: Settings, Integrations placeholder, E2E tests, and Polish.
